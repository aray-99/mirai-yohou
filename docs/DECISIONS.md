## [0001] リモートと CI/PR 運用

- 日付: 2026-07-07
- 論点: §0.5.3 は feature → develop の PR 運用を要求するが、本プロジェクトは自律実行で人間レビューを介さない。リモートの有無・PR の形式が未規定だった。
- 選択肢: (a) ローカルのみで運用し PR を省略 (b) GitHub リモートを作成し gh で PR を作る (c) リモートは作るが PR 相当の情報は `--no-ff` マージコミット body に記載する
- 採用: ユーザーが事前に作成していた既存の空リモート `aray-99/mirai-yohou`(public)をそのまま origin として利用(private 化を試みたところリポジトリが一時 disabled になったため、ユーザー了承のうえ public で運用)。main/develop を push し GitHub Actions CI(§0.5.4)を有効化。PR 必須記載事項(マイルストーン番号・通過テスト名・DECISIONS ID)は develop への `--no-ff` マージコミット body に記す (c)。
- 理由: §0.5.3 は「PR 本文またはマージコミット body」への記載を許容している。自律実行では gh PR を挟むよりマージコミット body の方が確実で、CI ゲートは push 時に機能する。
- 関連: §0.5.2 / §0.5.3 / §0.5.4

## [0002] Julia バージョン(ローカル 1.12 と CI 行列)

- 日付: 2026-07-07
- 論点: 仕様は Julia 1.10+ を要求(§0/§0.5.4)。ローカルは 1.12.6 であり、1.12 で生成した Manifest.toml は 1.10 の CI で julia_version 不一致となる。
- 選択肢: (a) CI を 1.12 のみにする (b) CI 行列 ['1.10', '1'] とし、1.10 側では Manifest を無視して resolve させる (c) juliaup で 1.10 を入れて Manifest を 1.10 で生成
- 採用: (b)。CI 行列は '1.10' と '1'(最新)。Manifest.toml は開発機の 1.12.6 で生成しコミットする。1.10 ジョブでは `Pkg.test()` 前に resolve が走ることを許容する。
- 理由: §0.5.4 の「1.10 を最低ライン」を CI で維持しつつ、開発機の完全再現は Manifest(1.12)で担保。両立の最小コスト解。
- 関連: §0.5.4 / §0.5.7

## [0004] thinning における強度評価点の近似

- 日付: 2026-07-07
- 論点: Ogata thinning で格子点間の候補時刻における λ(X_t) の評価には格子点間の状態が必要(§10)。
- 選択肢: (a) 格子点間を補間 (b) 直前格子点の状態で近似

- 理由: §10 の注意書きの指示どおり。Phase B で JumpProcesses.jl の VariableRateJump に移行した際に統計的一致を検証する。

## [0005] 分岐比 n の定義と v 依存の自己励起

- 日付: 2026-07-07
- 論点: ジャンプ時の λ_e 増分は α0·(v/v0)^κ_alphav(§6.3)であり、v > v0 の領域では実効的な分岐比が名目値 n = α0/β を超え得る。§6.1 の安定条件 assert がどちらを指すか曖昧。
- 選択肢: (a) 実効分岐比(v の上限を仮定)で assert (b) v = v0 基準の名目値 n = α0/β で assert し、λ_bar を安全弁とする
- 採用: (b)。パラメータ構築時に n = α0/β < 1 を assert(仮値では n = 0.4)。
- 理由: §6.1 の定義式が n = alpha0/beta と明記しており、同節が「lam_bar は数値安全弁」と位置づけているため。v 依存の実効爆発は λ_bar = 50/年 のクリップで防がれる。
- 関連: §6.1 / §6.3 / §7

## [0007] ジャンプと EM ステップの演算子分割順序

- 日付: 2026-07-07
- 論点: §10 の擬似コードは格子区間内の thinning を規定するが、区間内ジャンプと Euler–Maruyama 更新の適用順序、および区間内での lam_e 減衰・状態変化の扱いが未規定。
- 選択肢: (a) 区間内でドリフト・拡散・ジャンプを厳密に時刻順に交互適用 (b) 区間 [t, t+dt) のジャンプを先に処理(受理のたびに Γ を適用し、以後の候補は更新後の状態で評価)し、その後 dt 全幅の EM ステップを行う
- 採用: (b)。区間内の lam_e 減衰と連続力学の進行は無視する(強度評価は直前状態、DECISIONS #0004 と同じ O(dt) 近似)。
- 理由: dt = 0.01 年は最速時定数 1/beta = 0.2 年より十分小さく、分割誤差は O(dt)。透明性・デバッグ容易性を優先する Phase A の方針(§10)に沿う。Phase B で JumpProcesses.jl の VariableRateJump と統計的一致を検証する。
- 関連: §6 / §10 / DECISIONS #0004

## [0008] regimes テストの「高確率」「ジャンプ ≈ 0」の定量化

- 日付: 2026-07-07
- 論点: §11 regimes テストは「安定国: 50年でジャンプ ≈ 0(高確率)/ 変動国: 少なくとも1回発火(高確率)。各20回反復で判定」と定めるが、「≈ 0」「高確率」が未定量。仮値での実測(シード1001〜1040の40反復)は、安定国: ジャンプ0回が33/40(82.5%)・平均0.45回(稀に自己励起連鎖で最大9回)、変動国: 全40回で1回以上(6〜24回、平均14.3回)。
- 選択肢: (a) 安定国は全20回でジャンプ0を要求(実測分布と矛盾し常に red) (b) 実測分布に余裕を持たせた閾値: 安定国はジャンプ0の反復が14/20以上かつ平均1回未満、変動国は1回以上発火が19/20以上
- 採用: (b)。テストは固定シード(決定的)で実行し、閾値は回帰検知のガードとして機能させる。
- 理由: lam0 = 0.02/年でも50年の基底期待値は約1回あり、文字通りの「全反復で0回」は仮値パラメータと整合しない。実測の二項分布(p≈0.83)に対し 14/20 は十分に緩い下限であり、かつレジーム分離(De<1 vs De>1)の定性は厳密に検証される。
- 関連: §7 / §11 / §12 M3

## [0009] 同化ドライバの実装詳細(src/assimilation.jl)

- 日付: 2026-07-07
- 論点: §9.2/§9.3 のハイブリッド同化ループの置き場所と、仕様が定めない細部: (i) 週次バッチ窓と dt=0.01年 グリッドの不整合(1週間 = 0.0192年)、(ii) 強制ジャンプ後の一時インフレーション 1.05 の適用期間、(iii) 観測時刻(月次 1/12年 等)とグリッドの不整合、(iv) §12 のディレクトリ構成に同化ドライバ用ファイルがない。
- 選択肢: (i) 窓境界でステップを分割 / 2グリッド(0.02年 ≈ 7.3日)で近似 (ii) 固定期間 / 次回解析まで (iii) ステップ分割 / 最近傍グリッドに丸め (iv) enkf.jl に同居 / 新規 src/assimilation.jl
- 採用: (i) 窓 = 0.02年(2グリッド)で近似 (ii) 強制ジャンプ後「最初の解析ステップ」のみ 1.05、以後 1.02(§9.3 の「広げ直す」目的に合致)(iii) 観測・イベント時刻は最近傍グリッドに丸め(誤差 ≤ dt/2 = 1.8日)(iv) 新規 src/assimilation.jl(EnKF 解析カーネルと駆動ループの関心分離。§12 の構成への追加であり既存ファイルの変更ではない)(v) ポアソン log 重みは窓ごとに破棄せず、再抽選まで累積する(標準的な適応リサンプリング粒子フィルタの解釈)。
- 理由: いずれも O(dt) 近似で Phase A の精度と整合。ドライバ分離は enkf_linear テストの純粋性(解析カーネル単体検証)を保つ。(v) は必須と判明: 窓ごとに重みを破棄すると、同化中は内生降伏がオフのため σ_s が暴走したメンバーが淘汰されず(1窓の重み差は微小)、強制ジャンプ時に m = (1-ρ)σ⁻ の巨大衝撃で発散する(E1a 初回ランで σ_s=90 のメンバーが xi_tau を発散させた)。累積重みは高 σ_s メンバー(Λ が観測カウント0と不整合)を数窓で淘汰し、これが §9.3 ハイブリッドの安定化機構そのものである。
- 関連: §9.2 / §9.3 / §10 / §12

## [0011] 病的メンバーの重みゼロ化ガード(数値安全弁)

- 日付: 2026-07-07
- 論点: 同化中は内生降伏がオフ(§9.3)のため、EnKF 更新・インフレーションで σ_s が暴走したメンバーが生じ得る。累積ポアソン重みは Λ の不整合でこのメンバーの重みを実質ゼロにするが、ESS = 1/Σw² は単一外れ値ではほぼ低下しない(w_i→0 は ESS を N 付近に保つ)ため再抽選が発火せず、外れメンバーが強制ジャンプ時に m = (1-ρ)σ⁻ の巨大衝撃で Inf/NaN を発生させ、EnKF の解析(Cyy の LU 分解)全体を落とす。E1a 初回ランで実際に発生(σ_s = 90 → xi_tau 発散。ジャンプ直後インフレーション 1.05 が引き金)。
- 選択肢: (a) logit 座標のクランプ(力学を歪める) (b) ESS 閾値を上げる(根本解決にならない) (c) 週次窓の評価時に「病的」メンバー(|logit ξ| > 15 または σ_s > e³ ≈ 20 または非有限)の log 重みを -Inf とし、1体でも該当すれば強制再抽選で置換する
- 採用: (c)。判定閾値は §9.4 の警告水準(|logit ξ| > 10)と降伏応力の正規化(σ_Y = 1)から十分外側に取り、正常なフィルタ動作(実測で σ_s ≤ 10 程度のメンバーは出現し得る)に干渉しない。
- 理由: 病的メンバーの Λ は観測イベントカウント0と大きく不整合であり、重みゼロ化はポアソン尤度の極限として §9.3 の意味論に整合する。状態のクランプと異なり力学・分布を直接歪めず、lam_bar と同種の「数値安全弁」(§6.1)である。
- 関連: §6.1 / §9.3 / §9.4 / DECISIONS #0009

## [0012] E1 不合格の診断(基準・パラメータは不変のまま報告)

- 日付: 2026-07-07
- 論点: §13 の双子実験 E1a/E1b が凍結しきい値(test/acceptance_thresholds.toml)で不合格。§13 の規定に従い、基準を動かさず診断を記録してユーザーに報告する。公式ラン(シード: 真値4501/観測4502/アンサンブル4503/自由4504/E1b 4505、コミットは本エントリ直前)の結果: RMSE比 k=18.2, g=0.69, tau=0.81, p=1.04(基準<0.5)、σ_s相関=0.40(>0.6)、被覆率=0.987(0.80–0.98)、tauA誤差 同化0.356 vs 自由0.040、ランクχ²=358.8(臨界16.9)、E1b theta_sig事後=発散。
- 診断(フィルタ崩壊か、可同定性不足か、実装バグか):
  - **主因はフィルタ崩壊(仕様の同化設定の構造問題)**: §9.2 の乗法インフレーション 1.02 を「解析ステップ毎」に適用すると、多レート観測(月次 v・四半期 y・年次ほか)では45年で約900回の解析があり、偏差が累積 1.02^900 ≈ e^17.8 倍に膨張する。強く観測される方向は解析で収縮するが、log y を通してしか観測されない k, h の直交方向(弱観測部分空間)は力学の収縮(≈6–15%/年)より速く膨張し、アンサンブル平均が漂流(xi_k 平均誤差 +2.2、最大メンバー +23)。感度実験(同一シード): ρ=1.0 で k比0.77/g比0.40/tau比0.40/p比0.91・σ_s相関0.81・被覆0.64、ρ=1.005 で k比0.92・被覆0.67、ρ=1.02 で k比18.2。発散が ρ に単調依存し、enkf_linear テスト(厳密KF一致)が green であることから実装バグは否定的。
  - **副因は可同定性不足**: p(分極)は2年毎 sd0.10 の観測に対し過程ノイズが支配的で、インフレーションなしでも RMSE 比 0.91(基準 0.5 は観測設計上ほぼ達成不能)。σ_s の被覆率はインフレーションなしで 0.64(過小分散)であり、「解析毎 1.02」と「なし」の間に適切な注入率が存在するが、その選定は §9.2 の数値(1.02)の変更を要する。
  - E1b の theta_sig 発散は主因の帰結(拡大パラメータ行が弱観測部分空間の膨張に巻き込まれる)。
- 対応: §13 の規定によりしきい値・パラメータの変更は行わない。想定される是正案(いずれも §9.2 の記述の解釈または改訂が必要): (a) インフレーションを「解析毎」でなく「単位時間あたり」に正規化(例: ρ_eff = 1.02^(Δt/0.25年))、(b) RTPS(事前スプレッドへの緩和)への置換、(c) 弱観測変数への付加的インフレーション、(d) p の基準の観測設計見直し。ユーザーの指示を待つ。
- 関連: §9.2 / §13 / §0.5.8 / DECISIONS #0009 / #0011
