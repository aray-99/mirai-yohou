## 0. このドキュメントの使い方(Claude Code への指示)

あなた(Claude Code)は、このドキュメントを唯一の仕様書(Single Source of Truth)として、
社会動態シミュレータ「未来予報」の第一段階(双子実験 E1 まで)を実装する。

**進め方の原則:**

1. 実装は §12 のマイルストーン M0 → M5 の順に進める。**各マイルストーンの完了条件
   (テスト green)を満たしてから次に進むこと。**
2. 仕様に曖昧さや矛盾を見つけたら、勝手に解釈せず `docs/DECISIONS.md` に
   「論点・選択肢・採用した判断・理由」を記録してから進む。
3. 数式は本仕様の記法(§2〜§6)を厳密に守る。変数のインデックス(§3)は固定であり、
   変更してはならない。
4. すべての確率的処理は乱数シードを明示的に受け取り、再現可能にする。
5. コミットはマイルストーン内の論理単位ごと。テストが red の状態でコミットしない。
6. バージョン管理・コミット・ブランチ・CI・成果物管理の運用は **§0.5 開発運用ルール**に
   厳密に従う。これは §12 のマイルストーン進行と一体の手続きであり、任意ではない。

**技術スタック:** Julia 1.10+ / SciML エコシステム
(DifferentialEquations.jl, JumpProcesses.jl, Distributions.jl, ほか §10 参照)

## 0.5 開発運用ルール(バージョン管理・CI・成果物管理)

本章は §0 の「進め方の原則」を手続き面で具体化するものであり、実装と同格の要求である。
このプロジェクトは(a)再現性、(b)検証可能性、(c)§13 の「合格基準を後から動かさない」
科学的厳密性、の3点を最優先する。以下のルールはすべてこの3点を守るために存在する。

### 0.5.1 Conventional Commits(必須)

コミットメッセージは `<type>(<scope>): <subject>` 形式に統一する。

- **type:** `feat` / `fix` / `test` / `refactor` / `docs` / `chore` / `perf` / `build` / `ci`
  から選ぶ。
- **scope:** モジュール名を使う。許容値は
  `coordinates` / `parameters` / `drift` / `diffusion` / `diagnostics` / `jumps` /
  `integrator` / `observation` / `enkf` / `weights` / `experiments` / `docs` / `ci`。
- **subject:** 命令形・現在形で簡潔に(72文字以内目安)。
- 数式・定式化の意図が変わる変更では、body に理由を記述し、対応する `DECISIONS.md`
  エントリを footer に `Refs: docs/DECISIONS.md#<ID>` として参照する。
- **支配方程式(§5)・状態インデックス(§3)・座標変換(§2)・合格基準(§13)**
  に後方非互換の変更を加える場合は、`BREAKING CHANGE:` footer を必須とする。

例:

```
feat(jumps): implement Hawkes self-excitation jump map Γ

除荷(xi_sig += log rho)・弾性衝撃(xi_tau)・塑性変形(xi_tauA)の
3種を §6.3 の表に厳密対応させて実装。分岐比 n<1 の assert は
parameters 側に置き、branching テストで担保。
Refs: docs/DECISIONS.md#0007-jump-map-sign
```

### 0.5.2 Git Flow(ブランチ戦略)

- **`main`:** 常にリリース可能(= 全テスト green)な状態のみを置く。直接コミット禁止。
- **`develop`:** 統合ブランチ。マイルストーン単位の作業合流先。
- **`feature/<milestone>-<topic>`:** 個別作業ブランチ。`develop` から切り、`develop`
  へマージする。命名例: `feature/M2-hawkes-jump`, `feature/M4-enkf-analysis`。
- **タグ:** 各マイルストーン(M0〜M5)完了で `develop` → `main` にマージし、
  `vM0` … `vM5` のタグを打つ。
- §0-5「red の状態でコミットしない」を feature ブランチ内にも適用する。作業途中の
  red コミットは、`develop` へマージする前に rebase で潰し、履歴上に残さない。

### 0.5.3 プルリクエストとマイルストーンのゲート化

feature → develop の各 PR(自律実行で人間レビューを介さない場合も含む)は、
以下を PR 本文またはマージコミット body に必ず記載する。

- 対応するマイルストーン番号(§12)
- 通過した §11 テスト名の一覧
- 追加・更新した `DECISIONS.md` エントリの ID

§12 の各完了条件をマージ条件(レビュー観点)として扱い、**完了条件のテストが
green でない PR はマージしない。** マイルストーンのゲートは §12 の表を唯一の基準とする。

### 0.5.4 CI(GitHub Actions)による再現性ゲート

フェーズ4で CI/CD を組む前提(元設計)を前倒しし、M0 の時点で最小構成を用意する。

- `main` / `develop` への push および PR で
  `julia --project -e 'using Pkg; Pkg.test()'` を実行する。
- 特に以下は毎回 CI で検証し、形骸化を防ぐ:
  - `reproducibility` テスト(同一シードで完全再現、§11)
  - `branching` テスト(n = alpha0/beta < 1 の assert、§6.1/§11)
  - `coordinates` の round-trip(§2/§11)
- Julia のバージョン行列は 1.10 を最低ラインとして固定する。

### 0.5.5 DECISIONS.md のフォーマット固定

§0-2 で参照される `DECISIONS.md` は grep・相互参照を可能にするため、1エントリを
次の体裁で統一する。

```
## [0007] ジャンプ写像 tauA の符号
- 日付: YYYY-MM-DD
- 論点: (何が曖昧・矛盾だったか)
- 選択肢: (検討した案)
- 採用: (採った判断)
- 理由: (なぜ)
- 関連: コミット <sha> / §6.3 / §14-4
```

ID(連番、ゼロ詰め4桁)は Conventional Commits の `Refs:` footer から一意に辿れる
ようにする。

### 0.5.6 実験ランのトレーサビリティ(成果物の再現性)

§13 の合格基準は「後から動かさない」と定めているため、実験成果物には来歴を刻む。

- 双子実験(§13)の真値ラン・同化ラン・自由ランを生成した際は、出力ファイルおよび
  `experiments/figures/` の各成果物に、**コミット SHA・乱数シード・使用パラメータ
  セット名(安定国/変動国)・生成日時**をメタデータ(サイドカー JSON か画像埋め込み)
  として付す。
- これにより「どのコードの・どのシードで出た予報円か」が常に一意に遡れる状態を保つ。
- E1a / E1b の合格判定を出した状態には `experiment/E1a-pass` 等のタグを打つ。

### 0.5.7 リポジトリ資産管理(.gitignore / Manifest / 大容量物)

M0 の時点で以下を確定し、`DECISIONS.md` に記録する。

- **`Manifest.toml` はコミットする**(依存の完全固定 = 再現性のため)。
- アンサンブル計算の生出力(バイナリ)・大容量の中間結果は Git 管理外にするか
  Git LFS を用いる。`figures/` の最終図は、サイズが小さければコミット可。
- `.gitignore` に計算スクラッチ・ログ・環境固有ファイルを列挙する。

### 0.5.8 合格しきい値の凍結(手続きによる担保)

§13 の「不合格でも基準を動かすの禁止」を技術的に守るため、合格しきい値
(RMSE 50%、時間相関 0.6、被覆率 80〜98% 等)を**宣言的データとして一箇所に
切り出す**。ファイルは `test/acceptance_thresholds.toml` とし、標準ライブラリの
`TOML`(追加依存なし)で `TOML.parsefile` により読み込む。

- 閾値は「コードではなくデータ」である。TOML にすることで(a)値の変更が1行 diff として
  可視化され §0.5.3 のレビューと噛み合う、(b)宣言的形式ゆえ閾値ファイルに任意ロジックが
  侵入せず「値だけを置く場所」という制約が形式で担保される、(c)CI サマリや将来の
  外部スクリプトから言語非依存で参照できる。
- 他の閾値からの**導出値**(例: 被覆率上限を下限から算出する等)は TOML には基の値のみを
  置き、導出はテスト側のコードで行う。TOML に計算式は書かない。
- この閾値ファイルの変更は、必ず `DECISIONS.md` エントリ(論点・理由)を伴う
  `docs` または `test` コミットとして行い、単独では変更しない。
- E1 が不合格の場合の対応は §13 の規定(基準を動かさず、診断を記録してユーザーに
  報告)に従う。しきい値の緩和で「合格させる」ことは、この節によっても禁止される。

## 1. システム概要(1分要約)

- 社会を 4 モジュール(人口・経済・技術・制度)/ **13 次元の状態ベクトル**で表す。
- 時間発展は **確率微分方程式(SDE)+ 状態依存ジャンプ過程(Hawkes 型)**。
  ジャンプは「社会的応力の降伏 → 除荷と塑性変形」という計算力学のメタファーで定式化。
- 現実データは **アンサンブルカルマンフィルタ(EnKF)+ ポアソン重み(粒子フィルタ的補正)**
  のハイブリッドで同化する。イベントデータ(政変・騒乱)は「検知」ではなく
  「ジャンプ時刻の条件付け(全メンバー強制ジャンプ)」で扱う。
- 出力は点予測ではなく **アンサンブルによる確率的予報円**。
- 第一段階のゴールは実データではなく、**双子実験(OSSE): 合成データからの真値復元**。

## 2. 記法と座標変換

### 2.1 設計原則

- 有界変数(0〜1)は logit 座標、正値変数は log 座標で状態を保持する。
  これにより (a) EnKF のガウス摂動で範囲制約が壊れない、(b) 主要な観測演算子が
  ほぼ線形になる、(c) ジャンプ写像が全て「加算」で書ける。
- **力学(SDE)は変換後の座標で直接定義する。** 自然座標で定義してから変換すると
  伊藤の補題による補正項の管理が必要になるため、原則としてこれを避ける。
  唯一の例外は σ_s(§5 式11)で、自然座標の物理式(Maxwell 粘弾性)から変換した式を
  伊藤補正込みで明示してある。実装はその最終形をそのまま使えばよい。

### 2.2 変換の定義

```
logit(x) = log(x / (1 - x)),   inv: sigmoid(z) = 1 / (1 + exp(-z))
log / exp は自然対数
softplus(z) = log(1 + exp(z))   (常に > 0。単調性の保証に使う)
(a)_+ = max(a, 0)
```

実装: `src/coordinates.jl` に `to_state / from_state`(全13変数の一括変換)と
変数別の変換を用意し、round-trip テストを書く(§11)。

## 3. 状態ベクトル X(t) ∈ R^13(インデックス固定)

| # | 記号 | 内容 | 保持座標 | モジュール | ジャンプ影響 |
|---|------|------|----------|-----------|:---:|
| 1 | xi_P | 総人口 P | log | M1 人口 | — |
| 2 | xi_w | 生産年齢人口比率 w | logit | M1 | — |
| 3 | xi_h | 人的資本指数 h | log | M1 | — |
| 4 | xi_k | 一人当たり資本 k | log | M2 経済 | ✓ |
| 5 | xi_g | 格差指標 g(ジニ相当) | logit | M2 | ✓ |
| 6 | xi_T | 技術フロンティア T | log(単調非減少) | M3 技術 | — |
| 7 | xi_phi | 技術普及率 φ | logit | M3 | — |
| 8 | xi_v | 情報伝播速度 v | log | M3 | — |
| 9 | xi_tau | 制度信頼 τ | logit | M4 制度 | ✓ |
| 10 | xi_tauA | 信頼のアンカー τ*(logit空間の値) | そのまま | M4 | ✓(塑性) |
| 11 | xi_sig | 社会的応力 σ_s | log | M4 | ✓(除荷) |
| 12 | xi_p | 分極度 p | logit | M4 | — |
| 13 | lam_e | Hawkes 興奮度 λ_e | 自然座標(≥0) | M4 | ✓(自己励起) |

注意点:

- xi_tauA は「logit 空間上のアンカー値」であり、それ自体は非有界。変換不要。
- lam_e は力学的に非負が保たれるが、EnKF の解析更新後に負になり得るため
  **解析ステップ直後に `max(lam_e, 0)` でクランプする**(§9.4)。
- σ_Y(降伏応力)= 1 に正規化済み。σ_s は常に「降伏までの割合」として読む。

## 4. 診断変数(状態には含めない。毎時刻、代数的に計算)

```
A(T, phi)  = A0 * T^theta_T * exp(theta_phi * phi)          # 全要素生産性
y          = A * k^alpha * (h * w)^(1 - alpha)              # 一人当たり産出
Y          = y * P                                          # 総産出
lam(t)     = min(lam_bar, lam_b(X) + lam_e)                 # ジャンプ強度(§6)
dep_ratio  = (1 - w) / w                                    # 従属人口比率
```

**ŷ(産出成長率)の計算規則 — 重要:**
ŷ = d log y / dt は「ノイズを除いたドリフト部分」として解析的に合成する
(数値微分は使わない。ノイズを微分すると発散するため)。

```
yhat = theta_T * mu_T                                       # 技術寄与
     + theta_phi * (dphi の自然座標ドリフト)                  # ≈ theta_phi * phi(1-phi) * drift(xi_phi)
     + alpha * drift(xi_k)
     + (1 - alpha) * ( drift(xi_h) + (1 - w) * drift(xi_w) )
```

循環依存はない(§5 の設計で、各ドリフトは y の「水準」にのみ依存し、
ŷ に依存するのは σ_s の載荷則だけになるよう切ってある)。

## 5. 支配方程式(全13本)

dW はすべて独立な標準ウィーナー過程。dN は強度 λ(t) の点過程(§6)。
ジャンプ時の更新は §6.3 にまとめ、ここではドリフト・拡散のみ記す。

**外生入力(時間の既知関数):** b(t) 出生率, d(t) 死亡率, mig(t) 純移動率,
wbar(t) 人口構造ターゲット。双子実験では定数(§8.4)。

```
(1)  d xi_P   = [ b(t) - d(t) + mig(t) ] dt                                   # 拡散なし
(2)  d xi_w   = kappa_w * ( logit(wbar(t)) - xi_w ) dt + sigma_w dW
(3)  d xi_h   = gamma_h * ( 1 - h / hmax(y) ) dt + sigma_h dW
        hmax(y) = h0 * (y / y_ref)^chi                       # 所得→教育投資
(4)  d xi_k   = [ s * tau^theta_tau * (y / k) - delta - (b - d + mig) ] dt
                + sigma_k dW                                 # ソロー型+信頼摩擦
(5)  d xi_g   = [ eta_gT * mu_T - kappa_g * ( xi_g - mu_gbar ) ] dt
                + sigma_g dW                                 # SBTC 載荷+平均回帰
(6)  d xi_T   = mu_T dt ,   mu_T = a_T * softplus( c_T0 + eta_Ty * log(y / y_ref) )
                                                             # 拡散ゼロ。単調性は構成的に保証
(7)  d xi_phi = r_phi * (v / v0)^kappa_phiv * tau^kappa_phitau dt + sigma_phi dW
                                                             # ロジスティック成長は logit 座標で
                                                             # ほぼ定数ドリフトに帰着する
(8)  d xi_v   = kappa_v * ( xi_T + c_v0 - xi_v ) dt + sigma_v dW
(9)  d xi_tau = [ kappa_tau * ( xi_tauA - xi_tau ) - eta_taup * p ] dt + sigma_tau dW
(10) d xi_tauA = eps_A * ( xi_tau - xi_tauA ) dt             # 拡散なし。遅い移動平均
(11) d xi_sig = [ L(X) * exp(-xi_sig) - delta_sig - 0.5 * sigma_sig^2 ] dt
                + sigma_sig dW
        L(X) = eta_g * (g - g_c)_+ + eta_p * p - eta_y * (yhat)_+
        # 自然座標の Maxwell 型  d sigma_s = [L - delta_sig * sigma_s] dt + sigma_sig * sigma_s dW
        # を log 座標へ変換した式。-0.5 sigma_sig^2 が伊藤補正項。
        # log 座標のおかげで L < 0 でも sigma_s > 0 が自動的に保たれる。
(12) d xi_p   = [ eta_pg * ( g - g_c ) - kappa_p * ( xi_p - mu_p ) ] dt
                + sigma_p0 * (v / v0)^kappa_pv dW            # ★拡散項側の連成(v が分極ノイズを増幅)
(13) d lam_e  = - beta * lam_e dt                            # 純減衰。増加はジャンプ時のみ
```

### 5.1 連成の疎性(依存グラフ)

y と ŷ は「ハブ」となる診断変数であり、これを展開した実効的なヤコビアン密度は
**約28%**(48/169 成分)。ステップ検討時の見積もり(18%)より密だが、これは
ハブ経由の間接依存を数えた正直な値であり、ハブ構造ゆえ疎ソルバの恩恵は残る。
ヤコビアンの疎性パターンは手書きせず、Symbolics.jl の `jacobian_sparsity` で
自動検出すること(Phase B、§10)。

| 式 | 直接依存(自分以外) | 経路 |
|----|----|----|
| (1)(2) | なし | 外生 |
| (3) | y = f(T, phi, k, h, w) | 所得→教育 |
| (4) | y, tau | 生産関数+信頼摩擦 |
| (5) | mu_T = f(y) | 技術のスキル偏向 |
| (6) | y | 産出→R&D |
| (7) | v, tau | 普及速度 |
| (8) | T | 技術→インフラ |
| (9) | tauA, p | アンカー回帰−分極侵食 |
| (10) | tau | 遅い追従 |
| (11) | g, p, yhat(= ほぼ全域) | 載荷則 ★最重要連成 |
| (12) | g(ドリフト), v(拡散側) | 格差→分極 / SNS増幅 |
| (13) | なし(ジャンプのみ) | 自己励起 |

## 6. ジャンプ過程(Hawkes 型・Markov 埋め込み)

### 6.1 強度

```
lam(t)   = min( lam_bar,  lam_b(X_t) + lam_e(t) )
lam_b(X) = lam0 * exp( theta_sig * (sigma_s - 1) + theta_p * p + theta_v * log(v / v0) )
```

- sigma_s = 1(降伏応力)でちょうど lam0 *exp(theta_p*p + ...) 倍。
  theta_sig が「降伏の鋭さ」を制御する(→∞ で完全塑性的な鋭い閾値)。
- lam_e の力学は式(13)+ジャンプ時の加算(§6.3)。指数カーネル Hawkes の
  履歴積分と等価(Markov 埋め込み)であり、**rate 関数は現在の状態のみで計算できる**。
- **安定条件(パラメータ構築時に assert):** 分岐比 n = alpha0 / beta < 1。
  余裕をもって n ≤ 0.7 を推奨。lam_bar は数値安全弁。

### 6.2 マーク(イベント規模)

ジャンプ発生時、まず残留率 rho を引き、解放エネルギー m を計算:

```
rho ~ Beta(a_rho, b_rho)          # 残留率(0〜1)
m   = (1 - rho) * sigma_s^-       # 正規化解放エネルギー(sigma_s^- は発生直前の値)
```

### 6.3 ジャンプ写像 Γ(全て加算。発生直前の値に適用)

| 変数 | 更新 | 意味 |
|------|------|------|
| xi_sig | += log(rho) | 除荷(部分的応力解放) |
| xi_tau | -= c_tau * m | 弾性衝撃(信頼の即時低下) |
| xi_tauA | -= c_star * m | **塑性変形**(平衡点の恒久移動) |
| xi_g | -= c_g * m | 大変動による格差圧縮 |
| xi_k | -= c_k * m | 資本破壊 |
| lam_e | += alpha0 * (v / v0)^kappa_alphav | 自己励起(連鎖の種) |
| 他7変数 | 不変 | 技術ラチェット・人口慣性 |

### 6.4 ジャンプ生成器の 2 モード設計(必須インターフェース)

```julia
abstract type JumpMode end
struct EndogenousHawkes <: JumpMode end            # 内生: thinning 法で発火(純予測用)
struct ExogenousEvents  <: JumpMode                # 外生: 観測イベント時刻に強制発火(同化用)
    times::Vector{Float64}
end
```

同化時は ExogenousEvents で全アンサンブルメンバーのジャンプ時刻を同期させる
(多峰性の発生源を消す。確率性はマーク rho にのみ残る)。純予測時は EndogenousHawkes。
この切替が可能なように、ジャンプ処理は積分器から分離した関数として実装すること。

## 9. 観測モデルとデータ同化

### 9.1 観測演算子 H(双子実験用の合成観測。全て変換座標で定義)

| 観測 | 頻度 | H(状態 → 観測) | 観測ノイズ sd |
|------|------|------------------|----------------|
| log P | 年次 | xi_P | 0.002 |
| logit w | 年次 | xi_w | 0.01 |
| log y | 四半期 | log A + alpha*xi_k + (1-alpha)*(xi_h + log(sigmoid(xi_w))) | 0.01 |
| logit g | 5年毎 | xi_g | 0.02 |
| log T 代理(特許) | 年次 | xi_T | 0.05 |
| logit phi | 年次 | xi_phi | 0.02 |
| log v | 月次 | xi_v | 0.05 |
| logit tau(サーベイ) | 3年毎 | xi_tau | 0.15 |
| logit p | 2年毎 | xi_p | 0.10 |
| イベント数 N | 週次バッチ | ∫ lam dt(§9.3) | ポアソン |

log y の行だけ弱い非線形(logit 経由の w)を含むが、EnKF の暗黙の線形化で吸収できる。
実装上は H を「状態→観測空間の関数」として書けば十分(接線行列は不要。
EnKF はアンサンブル共分散で代用する)。

### 9.2 逐次・非同期同化(マルチレート)

- 観測をバッチせず、**届いた時刻に、届いた観測だけで解析ステップ**を実行。
- 実装: 観測時刻の集合を積分器の停止点(tstops 相当)にし、停止のたびに
  該当する観測のみで EnKF 解析(摂動観測法)を行う。
- **頻度差の罠への対策:** 日次系列は週次平均にバッチ化して R を再計算。
  アンサンブルスプレッドの維持は RTPS(relaxation to prior spread、alpha = 0.7)を
  解析ごとに適用: 解析後の成分別偏差スプレッドを解析前のスプレッドへ割合 alpha
  だけ緩和する。観測に拘束されない成分への注入はゼロであり、複利膨張が起きない。
  13〜15次元・N=100 なら局所化は不要。
  (v1.0 原案の「乗法的インフレーション rho_inf = 1.02 を解析後に常時適用」は、
  多レート観測下で弱観測部分空間を複利膨張させフィルタ崩壊を招くため改訂した。
  経緯と選定根拠: docs/DECISIONS.md#0012, #0013)

### 9.3 イベント同化(ハイブリッド戦略)

1. **イベント同期:** 観測イベント時刻では ExogenousEvents モードで全メンバー強制ジャンプ
   (マーク rho は各メンバー独立に引く)。内生発火は同化中オフ。
2. **無イベント尤度(粒子重み):** 各解析窓 [t1, t2] でメンバー i の重み
   w_i ∝ Poisson(N_obs | Λ_i),  Λ_i = ∫ lam_i dt(数値積分でトラッキング)。
   有効サンプルサイズ ESS = 1 / Σ(w_i²) が N/2 を下回ったら系統リサンプリング。
3. 強制ジャンプ直後はインフレーションを一時的に 1.05 へ引き上げ、
   マーク不確実性でアンサンブルを広げ直す。

### 9.4 解析ステップの仕様(自作。行列演算 ~20行)

- 摂動観測型 stochastic EnKF。N = 100。
- 解析後の後処理: lam_e = max(lam_e, 0)、|logit 座標| > 10 で警告ログ。
- 参考実装: EnsembleKalmanProcesses.jl(ただし本体は自作し、テストで検証する)。

## 10. 数値計算方針(2フェーズ)

**Phase A(M1〜M5 はこれで完遂する): 固定刻み Euler–Maruyama + Ogata thinning**

- dt = 0.01 年(≒3.65日)。最速時定数 1/beta = 0.2 年を十分解像する。
- 13次元 × N=100 × 50年(5000ステップ)は計算量的に自明。透明性・デバッグ容易性を優先。
- ジャンプ: Ogata の thinning 法。擬似コード:

```
t := 現在時刻
while t < t_next_grid:
    候補間隔 ~ Exponential(lam_bar);  t += 候補間隔
    if t >= t_next_grid: break
    u ~ Uniform(0,1)
    if u < lam(X_t) / lam_bar:      # 受理
        マーク rho を引き、ジャンプ写像 Γ を適用
```

  注意: lam(X_t) の評価には格子点間の状態が必要だが、dt が小さいので
  直前格子点の状態で近似してよい(誤差 O(dt)。DECISIONS.md に記録)。

**Phase B(E1 合格後の最適化。今回のスコープ外だが設計は壊さない):**

- JumpProcesses.jl の VariableRateJump + 適応 SDE ソルバ(SOSRI 等)へ移行し、
  Phase A と統計的に一致することを確認。
- ModelingToolkit.jl + Symbolics.jl `jacobian_sparsity` で疎ヤコビアン自動生成、
  stiff 対応ソルバとマルチレート化を検討。

アンサンブル並列は Threads(メンバー間は独立)。メンバーごとに乱数シード固定。

## 13. 双子実験 E1 プロトコル(第一段階のゴール)

**手順:**

1. **真値ラン:** 変動国セット + 固定シードで 45 年間の単一軌道を生成(ジャンプ込み)。
2. **合成観測の生成:** §9.1 の H・頻度・ノイズで観測系列を作る。真値ランのジャンプ時刻を
   「イベントカタログ」として保存(週次カウントも生成)。
3. **E1a(状態推定のみ):** パラメータは真値を既知として固定。初期アンサンブル N=100 を
   真値からずらした分布(各座標に sd 0.3 の摂動)から生成し、同化を実行。
4. **E1b(パラメータ同時推定):** theta_sig(+theta_p)を状態拡大で追加。
   初期値は事前分布からサンプル。
5. **対照実験:** 同化オフの自由ラン(同じ初期アンサンブル)。

**合格基準(事前確定。実装後に動かさない):**

- 観測に近い状態(xi_k, xi_g, xi_tau, xi_p): 同化ありの RMSE が自由ランの **50% 未満**。
- 隠れ状態 sigma_s: 真値との時間相関 **> 0.6**、かつアンサンブル 90% 区間の被覆率 80〜98%。
- 塑性変数 tauA: 大ジャンプ後 10 年窓での推定値が真値方向に更新されている
  (自由ランより誤差減少)。
- ランクヒストグラム(観測変数): 平坦性のカイ二乗検定で棄却されない(有意水準5%)。
- E1b: theta_sig の事後平均が真値の ±50% 以内に収束。

**E1 で不合格の場合:** パラメータや基準を動かして「合格させる」ことは禁止。
不合格の診断(フィルタ崩壊か、可同定性不足か、実装バグか)を DECISIONS.md に記録し、
ユーザーに報告して指示を仰ぐ。

## 14. 既知の限界・未決事項(実装時に「解決」しようとしないこと)

1. **人口モジュールの縮約:** McKendrick PDE を3区分コンパートメントに縮約済み。
   精密化はモジュール境界を保ったまま将来差し替える(状態次元が増えるため EnKF 設計に影響)。
2. **実データのヴィンテージ問題:** GDP は事後改定される。第一段階(双子実験)には無関係だが、
   実データ投入時は当時版データの扱いを別途設計する。
3. **GDELT/ACLED の報道バイアス:** イベント規模とマーク rho の対応付けは実データフェーズの主戦場。
   本仕様のスコープ外。
4. **tauA の符号:** 現仕様はジャンプで常に悪化(-c_star*m)。「改革成功で信頼上昇」の分岐は
   マーク依存の確率的符号への拡張候補として保留。
5. **遅延観測(ジニ係数の2年遅れ):** 双子実験では遅延なしで生成。固定ラグ平滑化(EnKS)は
   実データフェーズで判断。
6. **フロー観測の期央近似:** GDP は本来「四半期積算」の観測。第一段階は期央値で近似。
7. **歴史は1回しか起きない:** 実データでの「検証済み」主張には常に統計的限界の留保が付く
   (walk-forward + 複数国で擬似的に標本を稼ぐのが精一杯)。
